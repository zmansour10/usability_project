# Implementation Plan - AeroSight: Global Aviation Analytics Platform

We will design and build **AeroSight**, a production-quality, modular, and visually stunning Streamlit application leveraging the Airline passenger dataset in `data2`. The application will move beyond simple data plotting into a rich business intelligence tool with demographics, flight operations, automated data quality profiling, anomaly detection, heuristic AI-assisted insights, and a live passenger booking simulator.

## 1. Domain & Problem Space Analysis

Based on our exploratory data analysis, the folder `data2` contains three versions of the Airline Passenger dataset. We selected **`Airline Dataset Updated - v2.csv`** as the definitive source because:
1. It contains string-hash **Passenger IDs** (instead of standard sequential integers), which is realistic for global reservation systems.
2. It correctly decouples the **Pilot Name** from the **Passenger Name** (in earlier versions, they were identical).
3. It contains comprehensive records of **98,619 passenger bookings** in 2022.

### Critical Insights & Dataset Characteristics:
- **Passenger Bookings**: Each row represents a passenger booking record for a flight departing from a specific origin airport (named `Airport Name` with IATA code `Arrival Airport`), flying in 2022 under `Pilot Name`, with a given `Flight Status` (On Time, Cancelled, Delayed).
- **Date Complexity**: Departure dates are stored in **mixed formats** (some `m/d/YYYY` and others `m-d-YYYY`/`d-m-Y`). Flexible mixed parsing is required.
- **Pilot Load**: The dataset contains 98,605 unique pilots for 98,619 flights. Almost all pilots fly exactly once, with only 14 pilots flying twice.
- **Uniformity (Synthetic Data)**: The delay and cancellation rates are perfectly uniform across all continents (~33.3% each). This indicates synthetic generation, which we will highlight in the **Data Quality & Anomaly Detection** tab as a data profiling insight.
- **Rich Demographics**: Passenger ages cover a uniform range from 1 to 90 years. We classify them into Child, Youth, Adult, and Senior.

---

## 2. Streamlit App Architecture

We will implement a clean, highly modular architecture that separates data loading, business logic, reusable UI widgets, and individual dashboard views.

```mermaid
graph TD
    App[app.py - Main Entrypoint & Router] --> Service[core/data_service.py - Caching & Loading]
    App --> UI[components/ui.py - Glassmorphism, CSS, Cards]
    App --> CoreSchema[core/schema.py - Schema Inference & Bugfix]
    App --> Analytics[core/analytics.py - Anomaly, Profiling & AI Insights]
    
    App --> View1[views/overview.py - Global Analytics & Choropleth Map]
    App --> View2[views/demographics.py - Passenger Age, Gender, Nationality]
    App --> View3[views/operations.py - Pilot Workload & Continent Performance]
    App --> View4[views/anomalies.py - Data Profiling, Outliers & Heuristics]
    App --> View5[views/data_explorer.py - Table Search, Export & Booking Simulator]
```

### Module Breakdown:
1. **`app.py`**: Handles routing, page configurations, global filters, and sidebar navigation using premium UI components.
2. **`core/data_service.py`**: Manages reading `Airline Dataset Updated - v2.csv` from `data2/`, caching with `st.cache_data`, performing Mixed-Date conversion, and handling updates from the simulator.
3. **`core/schema.py`**: Dynamically infers schemas and fixes the `TypeError` where `max_v` was used instead of `max_val` in the dataclass instantiation.
4. **`core/analytics.py`**: Implements metrics calculation, trend analysis, anomaly detection (e.g. pilots with too many cancellations, extreme age distributions), and heuristic-based business advice generator (AI insights).
5. **`components/ui.py`**: Defines professional HSL colors, modern glassmorphic cards, custom metric boxes, custom HTML/CSS injections, and custom alerts.
6. **`views/`**:
   - `overview.py`: Global Map (Plotly Choropleth), booking metrics, flights status breakdown.
   - `demographics.py`: Nationality patterns, age categorizations, and gender segmentations.
   - `operations.py`: Flight cancellations, busiest routes, monthly volumes, pilot schedules.
   - `anomalies.py`: Data health analysis, date format analysis, operational anomaly logs, and AI-generated insights.
   - `data_explorer.py`: Interactive search, data downloader, and a **Live Flight Booking Simulator** to create new records, validate them, and reload the dashboard.
